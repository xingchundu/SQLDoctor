# 索引优化规则（检索用）

## 规则 1：等值谓词优先左前缀

对 `WHERE a=? AND b=?` 的常见模式，复合索引列顺序应与 **等值在前、范围在后** 一致，例如 `(a,b)` 优于盲目 `(b,a)`（需结合选择性评估）。

## 规则 2：覆盖索引降低回表

若 SELECT 列可被某索引完全覆盖，可减少回表；注意维护成本与写入放大。

## 规则 3：排序与索引方向

`ORDER BY` 与 `WHERE` 同向时，考虑包含排序列的索引以避免 filesort；大数据量下 filesort 易成为瓶颈。

## 规则 4：隐式类型转换

比较两侧类型不一致（如数值列与字符串字面量）可能导致索引失效，应保持类型一致或使用显式转换在可索引形式上。

## 规则 5：统计信息与直方图

长期运行后若计划明显偏离实际，应更新统计信息（MySQL ANALYZE、PG ANALYZE、Oracle 统计收集），避免错选全表扫描。

## 规则 6：前缀索引与长文本

对 VARCHAR 极长字段，必要时使用前缀索引；注意选择性不足会导致扫描量增加。
